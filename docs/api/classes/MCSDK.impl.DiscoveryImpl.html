startAutomatedOperatorDiscovery()

startAutomatedOperatorDiscovery(string  $clientId, string  $clientSecret, string  $discoveryURI, string  $redirectURL, \MCSDK\discovery\DiscoveryOptions  $specifiedOptions, \MCSDK\discovery\IDiscoveryResponseCallback  $callback, array  $currentCookies)

Start the operator discovery process

If the operator cannot be identified the function will return the 'operator selection' form of the response. The application can then determine how to proceed i.e. open the operator selection page separately or otherwise handle this.

The operator selection functionality will display a series of pages that enables the user to identify an operator, the results are passed back to the current application as parameters on the redirect URL.

Valid discovery responses can be cached and this method can return cached data.

Parameters

string $clientId

the username for mobile connect discovery authorisation, developer credentials are specified on the GSMA developer portal
string $clientSecret

the password for mobile connect discovery authorisation, developer credentials are specified on the GSMA developer portal
string $discoveryURI

the URI to use for mobile connect discovery rest calls
string $redirectURL

the URL to redirect to after an operator has been discovered
\MCSDK\discovery\DiscoveryOptions $specifiedOptions

a verbose list of options used to derive the options to be used
\MCSDK\discovery\IDiscoveryResponseCallback $callback

a callback class to complete the discovery request
array $currentCookies

list of cookies used to maintain a rest based session

Throws

\Exception|\MCSDK\utils\RestException

Tags

getOperatorSelectionURL()

getOperatorSelectionURL(string  $clientId, string  $clientSecret, string  $discoveryURI, string  $redirectURL, \MCSDK\utils\TimeoutOptions  $specifiedOptions, \MCSDK\discovery\IDiscoveryResponseCallback  $callback)

Get a list of links specific to the discovered operator. These links allow us to communicate directly with an operator and authorise the details specified at the start of the discovery process, for instance MSISDN.

This will not reference the discovery result cache.

The returned URL will contain a session id created by the discovery server. The URL must be used as-is.

parseDiscoveryRedirect()

parseDiscoveryRedirect(string  $redirectURL, \MCSDK\discovery\IParsedDiscoveryRedirectCallback  $callback)

Uses the get params passed to our post discovery redirect URL to derive the mcc, mnc and subscriber id. These are poked into the ParsedDiscoveryRedirect object which is handed over to the callback class to finalise the response handling.

The function will parse the redirectURL and parse out the components expected for discovery i.e.

  • selectedMCC

completeSelectedOperatorDiscovery()

completeSelectedOperatorDiscovery(string  $clientId, string  $clientSecret, string  $discoveryURL, string  $redirectURI, string  $selectedMCC, string  $selectedMNC, \MCSDK\discovery\CompleteSelectedOperatorDiscoveryOptions  $specifiedOptions, \MCSDK\discovery\IDiscoveryResponseCallback  $callback, array  $currentCookies)

Handles the request to select an operator manually in the event the MSISDN could not be found or was invalid.

In the case there is already a discovery result in the cache and the Selected-MCC/Selected-MNC in the new request are the same as relates to the discovery result for the cached result, the cached result will be returned.

If the operator cannot be identified by the discovery service the function will return the 'operator selection' form of the response.
